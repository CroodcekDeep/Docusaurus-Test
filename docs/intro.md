Bienvenido al **Centro de Documentación Técnica** de Banco Pichincha. Este portal centraliza las guías, estándares y recursos técnicos para todos los equipos de desarrollo.

## ¿Qué encontrarás aquí?

| Sección | Contenido |
|---------|-----------|
| **Guías de Desarrollo** | Estándares de código, flujo de trabajo con Git y buenas prácticas |
| **Frontend** | Documentación de frameworks como React y Angular |
| **Arquitectura** | Patrones de diseño, microservicios y decisiones técnicas |
| **Infraestructura** | Despliegue en Azure, autenticación y configuración de entornos |

## Inicio rápido

1. Navega por la **barra lateral** para explorar las secciones
2. Usa la **barra de búsqueda** para encontrar temas específicos
3. Cada documento incluye ejemplos de código y referencias

## ¿Cómo contribuir?

Esta documentación es mantenida por los equipos de desarrollo. Si deseas agregar o actualizar contenido:

1. Clona el repositorio del proyecto
2. Crea o edita archivos Markdown en la carpeta `docs/`
3. Envía un Pull Request para revisión

:::tip
Los documentos usan **Markdown** con soporte para bloques de código, tablas, admonitions y más. Consulta la [guía de Docusaurus](https://docusaurus.io/docs/markdown-features) para conocer todas las funcionalidades disponibles.
:::
